# LexBharat Governance

## Purpose
This document defines how LexBharat evolves while preserving its law-first principles.

## Decision Process
- Major architectural changes require an RFC.
- Accepted RFCs are recorded as ADRs.
- Standards are updated only through documented review.

## Engineering Principles
1. Law before AI.
2. Standards before implementation.
3. Verify before release.
4. Every change must be testable.
5. Maintain backward compatibility whenever practical.

## Versioning
- Standards and code evolve independently.
- Breaking changes require a migration plan.
