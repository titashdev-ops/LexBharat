# Contributing to LexBharat

Thank you for contributing to LexBharat.

## Guiding Principles
- Law before AI.
- Standards before implementation.
- Verify every legal assertion.
- Keep changes small, reviewable, and well documented.

## Contribution Workflow
1. Open an RFC for significant architectural changes.
2. Discuss and refine the proposal.
3. Record accepted decisions as ADRs.
4. Implement with tests and documentation.
5. Submit a pull request for review.

## Coding Standards
- Write clear, maintainable code.
- Add tests for new functionality.
- Preserve backward compatibility where practical.
- Document public APIs and standards.

## Commit Philosophy
Each commit should improve the trustworthiness, clarity, or maintainability of LexBharat.
